# TG WS Proxy

**TG WS Proxy** — это локальный **MTProto-прокси** для **Telegram Desktop**, созданный для ускорения работы клиента путём перенаправления сетевого трафика через **WebSocket-соединения**.

Передаваемые данные продолжают использовать оригинальное шифрование Telegram. Для работы приложения не требуется подключение к внешним прокси-серверам, сторонним узлам или дополнительной инфраструктуре.

## Windows: быстрый запуск

Перейдите в [раздел релизов](https://github.com/Flowseal/tg-ws-proxy/releases) и загрузите подходящую сборку:

* `TgWsProxy_windows.exe` — Windows 10+ x64
* `TgWsProxy_windows_arm64.exe` — Windows 10+ ARM64
* `TgWsProxy_windows_7_64bit.exe` — Windows 7 x64
* `TgWsProxy_windows_7_32bit.exe` — Windows 7 x32

После первого запуска приложение откроет инструкцию по подключению Telegram Desktop. После завершения настройки программа будет работать в фоне и отображаться в системном трее.

### Управление через трей

В меню программы доступны следующие действия:

* **Открыть в Telegram** — автоматическое добавление прокси через ссылку формата `tg://proxy`
* **Скопировать ссылку** — копирование ссылки для самостоятельного подключения
* **Перезапустить прокси** — перезапуск прокси-сервиса без закрытия приложения
* **Настройки...** — изменение конфигурации через GUI (версия приложения, проверка обновлений GitHub)
* **Открыть логи** — просмотр журнала работы программы
* **Выход** — остановка прокси и завершение работы приложения

### Настройка Telegram Desktop

**Автоматическое подключение**

Кликните правой кнопкой мыши по значку TG WS Proxy в системном трее и выберите пункт **«Открыть в Telegram»**.

Если автоматическая настройка не сработала:

1. Откройте меню приложения и выберите **«Скопировать ссылку»**
2. Отправьте полученную ссылку в чат «Избранное» Telegram
3. Откройте ссылку и подтвердите подключение прокси

**Ручная настройка**

В Telegram Desktop откройте:

**Настройки → Продвинутые настройки → Тип подключения → Прокси**

Добавьте новое подключение:

* **Тип:** MTProto
* **Сервер:** `127.0.0.1` (или собственный указанный адрес)
* **Порт:** `1443` (или изменённый параметр)
* **Secret:** значение из настроек или лог-файлов
## Как работает TG WS Proxy

Общий принцип работы прокси:

```
Telegram Desktop → MTProto Proxy (127.0.0.1:1443) → WebSocket → Telegram DC
```

Процесс обработки соединения состоит из следующих этапов:

1. TG WS Proxy запускает локальный **MTProto-прокси**, который доступен по адресу `127.0.0.1:1443`
2. Приложение отслеживает сетевые подключения Telegram, направленные к IP-адресам серверов Telegram
3. Из первоначального **MTProto obfuscation init-пакета** определяется идентификатор используемого DC
4. После получения данных о DC создаётся защищённое **WebSocket-соединение (TLS)** с необходимым Telegram DC через доменные имена Telegram
5. Если соединение через WebSocket установить невозможно (например, сервер возвращает ответ `302`) — приложение автоматически переходит на использование **CfProxy** либо выполняет подключение через обычный TCP-канал

> [!IMPORTANT]
>
> ### Не загружаются фотографии или видео?
>
> **Перейдите в настройки прокси, откройте раздел DC → IP и оставьте только `4:149.154.167.220`, удалив остальные адреса.**
>
> **Если это не помогло — полностью очистите значение в поле DC → IP.**
>
> Подобная ситуация может возникать у пользователей Telegram-аккаунтов без активной подписки Premium.
>
> Если проблема сохраняется, выполните настройку собственного домена по инструкции: [CfProxy.md](./CfProxy.md)

## Автоматическая сборка

Для формирования готовых сборок проекта применяются конфигурации PyInstaller:

* [`packaging/windows.spec`](../packaging/windows.spec)
* [`packaging/macos.spec`](../packaging/macos.spec)
* [`packaging/linux.spec`](../packaging/linux.spec)

Кроме этого, в проекте используется GitHub Actions workflow:

* [`/.github/workflows/build.yml`](../.github/workflows/build.yml)

Workflow автоматически собирает приложение для всех поддерживаемых платформ.

Минимальные требования для доступных бинарных версий:

* `TgWsProxy_windows.exe` — Windows 10+ x64
* `TgWsProxy_windows_arm64.exe` — Windows 10+ ARM64
* `TgWsProxy_windows_7_64bit.exe` — Windows 7 x64
* `TgWsProxy_windows_7_32bit.exe` — Windows 7 x32
* Intel macOS — версия 10.15 или выше
* Apple Silicon macOS — версия 11.0 или выше
* Linux x86_64 — для работы значка в системном трее требуется AppIndicator
