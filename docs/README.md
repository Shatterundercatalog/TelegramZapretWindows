# TG WS Proxy

**TG WS Proxy** — это локальный **MTProto-прокси** для **Telegram Desktop**, который предназначен для ускорения работы клиента за счёт перенаправления соединений через **WebSocket-соединения**.

Передача данных выполняется с сохранением исходного шифрования Telegram. Для работы приложения не требуется использование внешних серверов, сторонних прокси или дополнительной инфраструктуры.

## Windows: быстрый запуск

Откройте [страницу релизов](https://github.com/Flowseal/tg-ws-proxy/releases) и скачайте соответствующую версию приложения:

* `TgWsProxy_windows.exe` — Windows 10+ x64
* `TgWsProxy_windows_arm64.exe` — Windows 10+ ARM64
* `TgWsProxy_windows_7_64bit.exe` — Windows 7 x64
* `TgWsProxy_windows_7_32bit.exe` — Windows 7 x32

При первом запуске программа покажет инструкцию для подключения Telegram Desktop. После завершения настройки приложение продолжит работу в фоновом режиме через системный трей.

### Функции меню трея

В меню управления доступны следующие возможности:

* **Открыть в Telegram** — автоматическое подключение прокси в Telegram с использованием ссылки `tg://proxy`
* **Скопировать ссылку** — сохранение ссылки для ручного подключения
* **Перезапустить прокси** — повторный запуск прокси без закрытия программы
* **Настройки...** — изменение параметров через графический интерфейс (версия приложения, проверка обновлений GitHub)
* **Открыть логи** — просмотр журнала работы приложения
* **Выход** — остановка прокси и полное закрытие программы

### Настройка Telegram Desktop

**Автоматическое подключение**

Нажмите правой кнопкой мыши по значку TG WS Proxy в системном трее и выберите пункт **«Открыть в Telegram»**.

Если автоматическое подключение не выполнено:

1. Откройте меню программы и выберите **«Скопировать ссылку»**
2. Отправьте полученную ссылку в чат «Избранное» Telegram
3. Откройте ссылку и подтвердите использование прокси

**Ручная настройка**

В Telegram Desktop перейдите:

**Настройки → Продвинутые настройки → Тип подключения → Прокси**

Создайте новое подключение с такими параметрами:

* **Тип:** MTProto
* **Сервер:** `127.0.0.1` (или указанный пользователем адрес)
* **Порт:** `1443` (или изменённый порт)
* **Secret:** значение из конфигурации или лог-файлов
## Как работает TG WS Proxy

Принцип работы прокси можно представить следующим образом:

```
Telegram Desktop → MTProto Proxy (127.0.0.1:1443) → WebSocket → Telegram DC
```

Основная последовательность действий приложения:

1. TG WS Proxy создаёт локальный **MTProto-прокси**, доступный по адресу `127.0.0.1:1443`
2. Программа анализирует подключения Telegram, которые направляются к IP-адресам серверов Telegram
3. Из стартового **MTProto obfuscation init-пакета** извлекается информация об используемом DC
4. После определения нужного DC устанавливается защищённое **WebSocket-соединение (TLS)** через Telegram-домены
5. Если установить WebSocket-соединение не удаётся (например, при получении ответа `302`) — приложение автоматически выполняет переключение на **CfProxy** или использует прямое TCP-подключение

> [!IMPORTANT]
>
> ### Не загружаются фото или видео?
>
> **Откройте настройки прокси и в разделе DC → IP оставьте только `4:149.154.167.220`, удалив остальные значения.**
>
> **Если это не решило проблему — полностью очистите поле DC → IP.**
>
> Подобное поведение может наблюдаться у пользователей Telegram-аккаунтов без Premium.
>
> Если проблема остаётся, выполните настройку собственного домена согласно инструкции: [CfProxy.md](./CfProxy.md)

## Автоматическая сборка

Для создания сборок проекта используются конфигурационные файлы PyInstaller:

* [`packaging/windows.spec`](../packaging/windows.spec)
* [`packaging/macos.spec`](../packaging/macos.spec)
* [`packaging/linux.spec`](../packaging/linux.spec)

Также проект содержит GitHub Actions workflow:

* [`/.github/workflows/build.yml`](../.github/workflows/build.yml)

Данный workflow автоматически выполняет сборку приложения для всех поддерживаемых операционных систем.

Минимальные требования для готовых бинарных сборок:

* `TgWsProxy_windows.exe` — Windows 10+ x64
* `TgWsProxy_windows_arm64.exe` — Windows 10+ ARM64
* `TgWsProxy_windows_7_64bit.exe` — Windows 7 x64
* `TgWsProxy_windows_7_32bit.exe` — Windows 7 x32
* Intel macOS — версия 10.15 или новее
* Apple Silicon macOS — версия 11.0 или новее
* Linux x86_64 — требуется наличие AppIndicator для корректной работы значка в системном трее
